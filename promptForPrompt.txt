## CRITICAL PROJECT CONSTRAINTS MUST BE ENFORCED IN PROMPTS

### Prototype baseline preservation
- Existing demo codebase must remain runnable at all times.
- Do not delete the prototype.
- Preserve the baseline flow continuously:
  - load sample data,
  - build SQLite database,
  - generate features,
  - score machine risk,
  - launch Streamlit dashboard.
- Maintain a frozen reference tag or branch: `prototype-v0`.

### Simulated data honesty
- The dataset is simulated.
- Do not claim real semiconductor factory accuracy.
- Any model metric, including 84 percent recall, must be labeled as simulated-data performance.

### No large AI model needed
- Do not download a large AI model.
- Do not use OpenAI or external LLMs for the ML pipeline.
- Use scikit-learn for predictive maintenance modeling.
- Recommended baseline: `RandomForestClassifier`.
- Use `joblib` for saving/loading local model artifacts.

### Storage
- Use SQLite for MVP persistence.
- Do not rely only on in-memory DataFrames.
- Store cleaned tables, features, predictions, model runs, and data quality issues in SQLite.
- Store model artifacts in `models/`.
- Store sample data in `data/sample/` or `data/raw/`.

### Streamlit UI
- Use Streamlit as the dashboard framework.
- Keep Streamlit UI code in `app/`.
- Keep reusable logic in `semipulse/`.
- Do not mix heavy data processing directly into page code.

### Reproducibility
- Use fixed random seed for sample data and model training.
- Make file paths configurable through environment variables.
- Include tests for validation, feature engineering, database helpers, and model pipeline.
