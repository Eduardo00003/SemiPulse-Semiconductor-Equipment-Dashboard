# Current State

Last updated: 2026-05-08

## Summary

SemiPulse is implemented as a complete local MVP for simulated semiconductor predictive maintenance. The app can generate sample data, rebuild SQLite, generate machine features, train and score a local scikit-learn model, launch the Streamlit dashboard, run through Docker Compose, and export dashboard data.

## Implemented Capabilities

- Project requirements and build history are documented in `MasterPrompt.md`, `TECHNICAL.md`, `README.md`, `START_HERE.md`, and `prompts/`.
- Runtime configuration is environment-driven through `semipulse/config.py` and `.env.example`.
- Simulated datasets are generated by `semipulse/sample_data.py`.
- SQLite schema and helpers exist in `db/schema.sql`, `semipulse/schema.py`, and `semipulse/database.py`.
- CSV validation, cleaning, and rebuild logic exist in `semipulse/validation.py` and `semipulse/data_loader.py`.
- Machine-level features are generated and persisted by `semipulse/features.py`.
- A local `RandomForestClassifier` model is trained by `semipulse/model.py`; predictions are stored by `semipulse/predict.py`; metrics are read by `semipulse/metrics.py`.
- `semipulse/pipeline.py` orchestrates the demo pipeline for CLI, tests, and Streamlit actions.
- Streamlit pages are implemented for Overview, Data Upload / Load, Machine Health, Maintenance Risk, Defect Trends, Downtime Analysis, Model Performance, and Data Explorer.
- `semipulse/exports.py` supports risk ranking CSV, machine features CSV, model metrics JSON, and selected table CSV exports.
- `Dockerfile`, `docker-compose.yml`, and `.dockerignore` support containerized Streamlit execution with Docker-managed runtime volumes.

## Known Issues and Limitations

- The dataset is simulated and model metrics do not represent real semiconductor factory performance.
- The latest simulated training split produced zero recall, so dashboard and docs must avoid implying production-grade predictive accuracy.
- SQLite is used for local MVP persistence, not production concurrency.
- Streamlit is sufficient for a portfolio/demo workflow but is not a production web architecture.
- Docker Compose uses named volumes for `/app/data`, `/app/db`, and `/app/models`; generated runtime state persists in Docker volumes rather than host folders.
